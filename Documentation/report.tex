\section{Part I}
% =============================================================================

\subsection{Data Overview}

\subsubsection{File size, class balance, scale of features}

\ \ The PartI\_dev.csv file contains 48 features. The dataset comprises 46,805 numerical samples evenly distributed across 11 distinct classes, with exactly 4,255 samples per class. This represents a perfectly balanced dataset, eliminating the need for class weighting or synthetic oversampling. There are no missing values.


\subsection{Design Decisions}
\ \ The large number of samples and perfectly balanced classes allowed for a straightforward evaluation of various models without the need for complex resampling techniques.

\subsubsection{Pipeline elements (Traditional Method)}

\ \ StandardScaler was utilized to normalize the features to zero-mean and unit-variance. Normalization ensures that models sensitive to feature scale, such as K-Nearest Neighbors and Logistic Regression, perform optimally.

\ \ For dimensionality reduction, Principal Component Analysis (PCA) was evaluated. A variance threshold of 95\% was applied to reduce the dimensionality of the feature space. Both the original raw features and the PCA-transformed features were tested to determine the most predictive input space.

\subsubsection{Pipeline elements (Deep Learning Method)}

\ \ Similar to the traditional approach, StandardScaler was used to normalize the raw features. Since the dataset is perfectly balanced, techniques like SMOTE or class-weighted loss functions were unnecessary. The deep learning model was trained directly on the normalized raw features.

\subsubsection{Model selection (Traditional)}

\ \ A variety of traditional machine learning models were benchmarked, including Logistic Regression, Random Forest, Gradient Boosting, K-Nearest Neighbors (KNN), Linear Discriminant Analysis (LDA), and Quadratic Discriminant Analysis (QDA). Additionally, a soft-voting ensemble combining Random Forest, Gradient Boosting, and LDA was evaluated. 

\ \ To ensure robust performance estimates, StratifiedKFold cross-validation with 5 splits was used to compare the models on the raw features. The models were evaluated primarily on classification accuracy and macro F1-score.

\subsubsection{Model selection (Deep Learning)}

\ \ A Multi-Layer Perceptron was selected for the deep learning approach. The network architecture consists of two hidden layers with 128 and 64 neurons, respectively. BatchNorm was included after each linear layer to normalize activations and stabilize training. ReLU was utilized as the activation function, along with Dropout(0.3) to prevent overfitting. 

\ \ The model was compiled using CrossEntropyLoss, and optimized using Adam with a weight decay of 1e-4. Model performance was validated using 5-fold StratifiedKFold cross-validation.

\section{Part II}
% =============================================================================

\subsection{Data Overview}

\subsubsection{File size, class balance, scale of features}

\ \ The PartII\_dev.csv file contains 125 features. Each feature has 158 numerical samples, with 126 belonging to class 0 and 32 belonging to class 1. This represents a 4:1 class imbalance. The features have widely ranging scales and many feature pairs are correlated (|r|>0.90). There are no missing values.


\subsection{Design Decisions}
\ \ The massive redundancy in features and very small sample size were the main constraints for this part of the project. Four iterations were tested with the final being the selected iteration of the models.

\subsubsection{Pipeline elements (Traditional Method)}
 
\ \ StandardScalar was used in all versions of the traditional ML solution. StandardScalar normalizes the data to zero-mean and unit-variance, which was necessary due to the variety of scale in the dataset.

\ \ For dimensionality reduction, initially PCA was selected. PCA discards all features except for the ones that account for 95\% of the variance (in this case, all but 11 features). However, those high-variance features were not actually the most \underline{predictive} features. Instead, SelectKBest was used. SelectKBest runs an ANOVA F-test on each feature independently against Y. The 10 most statistically discriminative features were kept and the rest were discarded.

\ \ In earlier versions, SMOTE was used to generate synthetic minority-class samples. However, it did not show significant improvements in performance. LogReg can be set to class\_weight(balanced) to adjust for the unbalanced classes instead.

\subsubsection{Pipeline elements (Deep Learning Method)}

\ \ SelectKBest and StandardScalar are again used for the deep learning model. Like the traditional method, SMOTE was originally used. Not only did it not improve performance, it seemed to actively degrade it. In the final version there is no minority class synthesis.

\subsubsection{Model selection (Traditional)}

\ \ To select a model, StratifiedKFold cross validation (specdifically RepeatedStratifiedKFold) with 5 splits repeated 5 times was used to compare models. CV F1, CV AUC-ROC, and threshold were compared between LogReg with SelectKBest(k=10), LogReg with SelectKBest(k=7), SVM using RBF kernel and SelectKBest(k=8), and LogReg with PCA. 

\begin{figure}[h]
  \centering
  \includegraphics[width=\textwidth]{figures/traditional_model_CV}
  \caption{Result of model selection StratifiedKFold cross validation}
\end{figure}
 
\ \ Based on the CV results, LogReg with SelectKBest(k=10) was selected as the traditional learning method.

\subsubsection{Model selection (Deep Learning)}

\ \ MLP (Multi-Layer Perceptron) was, based on research, used for this dataset. MLP is a deep learning model well-suited for non-linear class boundaries. After experimenting with model size, it was determined to be best to use a smaller model. The original large model not only took too long to train, but was also not performant on the very small dataset.

\ \ The final model used a single hidden layer. BatchNorm was used to normalize activations to zero mean/unit variance per batch, which stabilized training and acted as a mild regularizer. BatchNorm also reduced sensitivity to weight initialization. ReLU was used as the activation function with Dropout(0.25). BCEWithLogitsLoss was used for the output layer. For the final training, the model trained for 250 epochs and used ensemble over 5 random initializations. The final model had 220 parameters.

\ \ To determine the best configurations, RepeatedStratifiedKFold CV was used for 15 folds. During these training steps, early stopping was utilized.

\begin{figure}[h]
  \centering
  \includegraphics[width=\textwidth]{figures/deep_model_CV}
  \caption{Result of config selection StratifiedKFold cross validation}
\end{figure}

\ \ Based on these results, the best configuration and the one used was k12\_16: 12 input features, 16 neurons in the hidden layer, 0.25 dropout (randomly zero 25\% of neurons, helps prevent overfitting), weight decay of 1e-12, and learning rate of 1e-3.
